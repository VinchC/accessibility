A few good practices regarding web accessibility

Including some dos and don'ts

Includes advice, tools, semantics and more !

Content description

Content must be described as precisely as possible, an "array" doesn't provide any information about the content.

Buttons

Buttons are CTAs and must not be used for navigation, only to interact with an element or submit a form.

It is important to be as accurate as possible regarding the text inside the CTA elements (Validate != Validate the form) and the links (link != link to website content page)

Screen readers don't understand this (div ? class ?)

div class="btn donts">Add - BAD PRACTICE

type="button" should be used whenever an action is performed on an element (including creation, animation)

type="submit" should be used to validate a form.

Use different id and labels to differentiate them from the others.

Semantics

Use as many HTML tags as possible and avoid div to optimize the tab navigation.

Allows the website to be browsed correctly by robots - which is a good practice for SEO.

Focus

Title tags

Use the correct hierarchy of the HTML tags (h1 (one only) > h2 (two max) > h3 (three or four) etc.)

Color contrast

Screen should be readable at all times, by everybody, whatever the time of day or night, light etc.

This website allows to grade the contrast between colors and lightness parameter.

Lighthouse Evaluation Tool

A tool accessible via the Web inspector that allows to analyze a website on some criteria including accessibility, performances, SEO and good practices in general.

Skip navigation link

Allows to skip the content of the menu to access directly to the page content.

Agendas

Agendas are tricky because they include a lot of element that must be accurately identified and clearly state the dates of beginning and end.

Pop up windows

If the pop-up is not designed correctly, it will be impossible for a screen reader to read it!

Forms

Forms may have a particular format expected for some fields (birthdate, phone number...), the error messages must be accurate and accessible to the user for each case.
